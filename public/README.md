# Public Analysis Materials

This folder contains the public code, anonymized data, and exported result tables for the paper.

## Reproducibility Status

This archive is reproducible in a practical sense, with one important distinction:

- The R-based statistical analyses can be rerun from the included scored CSV files in `data/`.
- The Python notebooks used to preprocess the anonymized exports are included, but they require a Python/Jupyter environment that is not bundled in the repository.
- The coded qualitative CSV files are included as fixed inputs. They are not regenerated by code in this archive.

## Folder Structure

```text
public/
├── export/                           # Anonymized first-study inputs and exported descriptive tables
├── export_followup/                  # Anonymized follow-up inputs and exported descriptive tables
├── data/
│   ├── firststudy/                   # Pre-generated scored CSVs for first-study regressions/tests
│   └── followup/                     # Pre-generated scored CSVs for follow-up regressions/tests
├── analysis.ipynb                    # First-study descriptive analysis notebook
├── analysis-followup.ipynb           # Follow-up descriptive analysis notebook
├── regression_export.ipynb           # First-study preprocessing notebook for regression inputs
├── regression_followup.ipynb         # Follow-up preprocessing notebook for regression inputs
├── *.Rmd                             # Statistical analysis scripts
├── run_r_analyses.R                  # Helper script to run all R analyses without Pandoc
├── requirements.txt                  # Python dependencies for the notebooks
├── surveyfields.py                   # First-study field definitions
├── surveyfieldsfollowup.py           # Follow-up field definitions
└── *.csv                             # Exported result tables already generated for the paper
```



## Full Workflow

### Stage 1: Optional Python notebooks

The notebooks are needed only if you want to rebuild the scored CSV files in `data/` from the anonymized exports in `export/` and `export_followup/`.

Install the Python dependencies:

```bash
cd public
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements.txt
```

Then run the notebooks in roughly this order:

| Notebook | Main input | Main outputs |
| --- | --- | --- |
| `analysis.ipynb` | `export/anonymized_data.csv` | `export/demographics.csv`, `export/scores.csv`, `export/learning_effect.csv` |
| `analysis-followup.ipynb` | `export_followup/anonymized_data.csv` | `export_followup/demographics.csv`, `export_followup/scores.csv` |
| `regression_export.ipynb` | `export/anonymized_data.csv` | `data/firststudy/*.csv` |
| `regression_followup.ipynb` | `export_followup/anonymized_data.csv` | `data/followup/*.csv` |

The `data/` outputs are already included, so Stage 1 is optional for most readers.

### Stage 2: R analyses

The R analyses consume the pre-generated files in `data/` and write result tables into `public/`.

Install the required R packages if needed:

```r
install.packages(
  c(
    "knitr", "ordinal", "lme4", "dplyr", "car", "sjPlot",
    "performance", "broom", "emmeans", "ggplot2", "MASS",
    "brant", "pscl", "multcomp", "RVAideMemoire", "purrr",
    "tibble", "EMT"
  ),
  repos = "https://cloud.r-project.org"
)
```
